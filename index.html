Proof of Concept

Custom Phish Reporter

A lightweight Outlook add-in that lets users report suspicious emails to their security team with one click — no authentication required.

How It Works

When a user clicks Report Phish in Outlook, the add-in opens a taskpane that reads the email details via Office.js and submits them to your API.

Report Phish
→
Step 2 Taskpane grabs
email + reporter
→
Step 3 POSTs .eml to

→
Step 4 Confirmation
shown to user

What Gets Submitted

The report payload is sent as JSON and includes everything your SOC team needs:

  • reporter — the mailbox owner's email (from Office.js userProfile)
  • reporterName — display name of the reporter
  • messageId — the internet message ID of the suspicious email
  • subject — email subject line
  • sender — the From address
  • receivedAt — timestamp the email was received
  • emlBase64 — the full original .eml file (base64 encoded)

No Authentication Required

Unlike the standard Mimecast add-in which requires users to log in, this PoC identifies the reporter using Office.context.mailbox.userProfile.emailAddress — which is provided by Outlook itself since the user is already signed in. No OAuth, no SSO, no login screen.
